# План сессии M10: Деплой на Hetzner VPS

**Назначение:** Точка входа для нового чата по деплою Riga Guide Bot на production-сервер.
**IP сервера:** `89.167.11.154` (Hetzner)
**Статус:** База знаний (KB) готова, протестирована и зелёная (18 уникальных мест). Системные скрипты для production (резервное копирование, cron, systemd) созданы. Все изменения закомичены.

---

## 1. Первичная настройка Hetzner VPS
1. **SSH-подключение:** Подключиться к `root@89.167.11.154` (или создать пользователя с sudo-правами).
2. **Базовая безопасность (Firewall):**
   - Установка `ufw`.
   - Разрешить SSH (порт 22). Запретить входящие по умолчанию.
   - Включить firewall: `ufw enable`.
3. **Установка Docker и Git:**
   - Обновить пакеты: `apt update && apt upgrade -y`.
   - Установить **Docker** и **Docker Compose** (через официальный репозиторий Docker).
   - Установить **Git**, `curl`, `tar`.
4. **Настройка Swap (Опционально, но критично):**
   - Если это минимальный тариф (2 GB RAM или меньше), добавить swap файл на 2 ГБ, чтобы избежать OOM Killer при запуске Docker контейнеров с Python:
   ```bash
   fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile
   ```

## 2. Перенос Проекта на VPS
1. **GitHub Clone:** 
   - Склонировать проект на сервере: `git clone <URL_РЕПОЗИТОРИЯ> ~/Riga_guide` (или использовать rsync/scp с локального компьютера).
   - Убедиться, что мы в нужной ветке `master`.
2. **Настройка переменных окружения:**
   - Создать файл `.env` в корне проекта на сервере.
   - Добавить ключи (локально скопировать): `TELEGRAM_BOT_TOKEN`, `GEMINI_API_KEY`, `TAVILY_API_KEY`.
3. **Перенос Базы Знаний (КРИТИЧНО!):**
   - Чтобы не парсить википедию и не тратить квоту заново, перенести локальную базу (Chroma и SQLite) на сервер.
   - Локально выполнить:
     ```bash
     scp -r data/chroma root@89.167.11.154:~/Riga_guide/data/
     scp data/bot.db root@89.167.11.154:~/Riga_guide/data/
     ```

## 3. Деплой Инфраструктуры бота
1. **Установка Systemd-службы:**
   - Перенести `infra/systemd/riga-guide.service` в `/etc/systemd/system/`.
   - Изменить пути в файле `.service`, чтобы они указывали на реальную директорию на VPS (`/root/Riga_guide` например).
   - Выполнить: `systemctl daemon-reload && systemctl enable riga-guide.service`.
2. **Установка Cron-задач (Rollup & Backup):**
   - Перенести/скопировать конфигурацию из `infra/cron/riga-guide.cron` в пользовательский crontab: `crontab -e`.
   - Убедиться, что скрипты `scripts/daily_rollup.py`, `scripts/backup_kb.sh` и `scripts/restore_kb.sh` имеют право на исполнение (`chmod +x`).

## 4. Запуск и Smoke-Test
1. **Запуск контейнера:**
   - Выполнить `systemctl start riga-guide.service` (или напрямую `docker compose up -d bot` для проверки логов).
   - Проверить статус: `docker compose logs -f bot`.
2. **Telegram Smoke Test:**
   - Отправить команду `/start` в бота (в Telegram).
   - Отправить тестовый текстовый запрос: *"Расскажи про Дом Черноголовых"*.
   - Отправить тестовое фото Домского собора.
   - Если бот отвечает корректно, latency в норме и нет ошибок — **Запуск успешен.**
