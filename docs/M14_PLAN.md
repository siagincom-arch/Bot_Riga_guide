## 2. Цель и не-цели M14

**Цель:** превратить бот из «расскажу-про-место» в «помогу-спланировать-визит». Добавить 4 категории контента и социальный сигнал качества (рейтинг).

**Не-цели (явно):**
- События — М15
- Локализация EN/LV — отдельная фаза
- Push/дайджест по утрам — Non-goal из `USER_SPEC §10`, остаётся
- Бронирование столиков, билетов — Non-goal
- Социальные функции: комментарии, обсуждения, профили — Non-goal
- TripAdvisor-стайл «топ-100 лучших» — только базовый `/top` с порогом ≥5 голосов

## 6. M14.3 — Рейтинг ⭐1–⭐3 (CRITICAL PATH)

**Цель:** социальный сигнал качества — пользователи оценивают рассказы и места.

### 6.1 Схема SQLite

```sql
-- Новые таблицы (миграция M14.3.1)

CREATE TABLE stories (
  story_id     TEXT PRIMARY KEY,        -- uuid4
  place_id     TEXT NOT NULL,
  chat_id      INTEGER NOT NULL,
  generated_at TIMESTAMP NOT NULL,
  summary      TEXT NOT NULL,           -- первый блок ответа (для /top)
  full_hash    TEXT NOT NULL,           -- sha256 полного текста, для дедупа
  source_type  TEXT NOT NULL            -- 'text' | 'photo' | 'geo' | 'followup' | 'route' | 'food'
);
CREATE INDEX idx_stories_place ON stories(place_id);

CREATE TABLE ratings (
  rating_id    INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id      INTEGER NOT NULL,
  target_type  TEXT NOT NULL,           -- 'story' | 'place'
  target_id    TEXT NOT NULL,           -- story_id или place_id
  stars        INTEGER NOT NULL CHECK(stars BETWEEN 1 AND 3),
  created_at   TIMESTAMP NOT NULL,
  updated_at   TIMESTAMP NOT NULL,
  UNIQUE(user_id, target_type, target_id)
);
CREATE INDEX idx_ratings_target ON ratings(target_type, target_id);

CREATE TABLE place_stats (
  place_id     TEXT PRIMARY KEY,
  avg_rating   REAL,
  votes_count  INTEGER NOT NULL DEFAULT 0,
  last_recalc  TIMESTAMP
);
```

### 6.3 Кнопки ⭐ — где появляются (двухступенчато)

| Сценарий | Шаг 1 (под ответом) | Шаг 2 (после тапа на «Оценить») |
|----------|---------------------|----------------------------------|
| Рассказ о месте (`status=ok`) | `[⭐ Оценить] [🎭 Ещё легенда] [📍 Что рядом]` | Сообщение «Понравился мой рассказ?» + `[⭐][⭐⭐][⭐⭐⭐]` |
| Карточка места (через `menu:top`) | `[💬 Рассказать] [📍 Что рядом] [⭐ Оценить место]` | «Понравилось это место?» + `[⭐][⭐⭐][⭐⭐⭐]` |
| Маршрут / Еда / Лайфхак / Транспорт | `[⭐ Оценить рассказ]` (только рассказ, не место) | «Понравился ответ?» + `[⭐][⭐⭐][⭐⭐⭐]` |

**Тексты вопросов бота при тапе «Оценить» (`i18n_ru.py`):**
- `RATE_STORY_PROMPT = "Понравился мой рассказ?"`
- `RATE_PLACE_PROMPT = "Понравилось само место?"`
- `RATE_GENERIC_PROMPT = "Понравился ответ?"` (для еды/маршрута/транспорта/лайфхака)

**Почему двухступенчато:**
1. Клавиатура под ответом не разрастается до 5 кнопок.
2. Бот **явно различает** «оценка рассказа» и «оценка места» через текст вопроса — пользователь не путается.
3. Промежуточное сообщение-вопрос даёт боту место объяснить, **что именно** оценивается.

### 6.4 Связь «рейтинг рассказа → рейтинг места»

**Решение:** независимые агрегаты, никакого влияния друг на друга. Причины:
- Рассказ оценивается за качество **текста** (бот). Место — за реальный **опыт визита**.
- Смешивание создаст шум: места с плохим текстом проседали бы без причины.
- Топ мест должен отражать **реальную ценность места**, не качество генерации.

Задача 14.3.4 **удалена** (была про «общий рейтинг рассказов влияет на place_stats»).

### 6.6 Формат `/top`

```
⭐ Топ мест Риги (по оценкам гостей):

1. Дом Черноголовых — ⭐⭐⭐ (24 голоса)
2. Турайдский замок — ⭐⭐⭐ (19)
3. Рижский замок — ⭐⭐⭐ (15)
4. Домский собор — ⭐⭐ (28)
5. Памятник Свободы — ⭐⭐ (12)

🎭 Топ рассказов бота:
1. Легенда о Кошкином доме — ⭐⭐⭐ (8)
...
[Полный список] (если ≥10 мест)
```

Минимум 5 голосов на место, 3 на рассказ — отсекает шум.

### 6.7 Карточка места (для `menu:top` → тап на место)

```
🏰 Рижский замок
⭐⭐⭐ 2.8 (15 оценок)
📍 Pils laukums 3, Vecrīga

[💬 Рассказать] [📍 Что рядом] [⭐ Оценить место]
```

Без рассказа сразу — пользователь сам решает, нужен ли текст. Экономит токены и не «бомбит» простыни.

### 6.8 Acceptance criteria

- Оценка ставится одним тапом, повторная — обновляет.
- `/top` показывает реальные данные после ≥5 голосов на место.
- Pытается оценить тот же рассказ дважды → второй UPSERT перезаписывает, `votes_count` не растёт.
- Юнит-тест `test_ratings.py` зелёный.

```
2. По каждому waypoint: 1-2 предложения о месте (берёшь из passages), время на остановку, переход к следующему.
3. В конце — end_tip.

НЕ повторяй полные легенды — это обзор маршрута, не рассказ о каждом месте.
НЕ используй маркеры списков. Текст должен читаться как речь гида.
Максимум 400 слов.
```

### 7.6 Acceptance criteria

- `/menu` → 🚶 Маршрут → «На сколько часов?» → выбор 2h → бот отдаёт связный текст маршрута на ~3 минуты чтения.
- Под ответом: «Расскажи подробнее про <первое место>» + ⭐.
- Маршруты ссылаются только на места из текущей KB (18 мест, см. PROTOCOL 2026-04-25).
